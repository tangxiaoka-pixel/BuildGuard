# BuildGuard 智慧工地演示系统

BuildGuard 是用于客户确认需求的本机演示系统，覆盖平台级、企业级、项目级上下文，以及实名制、安全教育、设备门禁、考勤、模拟上报和项目大屏闭环。

## 技术栈

- 前端：Vue 3、TypeScript、Vite、Element Plus、ECharts、Pinia、Axios
- 后端：Spring Boot 3、Java 17、Maven、Spring Data JPA
- 数据库：H2 文件数据库，文件位于 `backend/data/smart-site.mv.db`

项目自带本地 Java 17 和 Maven 环境，不依赖 Docker 或 Linux 虚拟机。

## 一键启动

```bash
./start-local.sh
```

需要关闭 Codex 后仍持续运行时，安装 macOS 用户后台服务：

```bash
./install-background-service.sh
```

后台服务由 `launchd` 托管，登录系统后自动启动，进程退出后自动重启。卸载：

```bash
./uninstall-background-service.sh
```

访问地址：

- 管理端：http://localhost:5173
- 现场入员 H5：http://localhost:5173/h5/worker-entry
- 后端 API：http://localhost:8082/api
- H2 控制台：http://localhost:8082/h2-console
- Swagger：http://localhost:8082/swagger-ui.html

停止服务：

```bash
./stop-local.sh
```

也可以分别启动：

```bash
cd backend
../.runtime/apache-maven-3.9.9/bin/mvn spring-boot:run

cd frontend
npm install
npm run dev
```

演示账号：

- 平台超级管理员：`admin / 123456`
- 单企业管理账号：`13800002001 / 123456`
- 多企业管理账号：`13800009999 / 123456`

## 账号与权限模型

- 平台超级管理员统一创建企业和管理账号。
- 管理账号可以授权一家或多家企业，只能访问被授权企业及其项目数据。
- 账号密码使用 BCrypt 哈希保存，登录后通过 Bearer Token 调用受保护接口。
- 租户仅作为内部兼容字段保留，不在管理后台展示，也不设置租户管理员。

## 推荐演示流程

1. 登录后查看平台工作台、企业和管理账号。
2. 点击“切换企业 / 项目”，选择“浙江华建工程有限公司”进入企业级。
3. 打开“项目健康度”，点击“进入项目”下钻到滨江项目。
4. 查看参建单位、项目区域、班组和劳务工人。
5. 新增工人，使用“模拟读取身份证”填充实名信息。
6. 查看教育课程、考题和考试记录，执行“模拟考试通过”，系统自动生成安全教育台账。
7. 给工人配置区域权限并下发设备；未通过安全教育时会被拒绝。
8. 在“模拟设备”生成今日模拟考勤，查看正常和异常记录。
9. 模拟人员、考勤上报并查看上报日志。
10. 打开“项目大屏”完成演示。

## 业务规则

- 人员安全教育状态以有效且已通过的安全教育台账为准。
- 设备权限下发要求：人员在场、安全教育已通过、存在有效区域权限。
- 模拟考勤会校验人员状态、安全教育和设备所属区域权限。
- 政府上报和设备连接均为 Mock，不接入真实外部系统。

## 现场入员 H5

- 仅项目管理员和已启用 H5 权限的班组长可以登录。
- 项目管理员可以在自己管理的项目中选择班组并新增劳务工人。
- 班组长使用工人手机号和单独设置的 H5 密码登录，只能向自己的班组新增人员。
- 在管理端“劳务工人”列表打开“班组长”开关并设置密码，即可启用班组长 H5 权限。
- H5 支持手机拍摄身份证和正面人脸；当前身份证识别为模拟 OCR，真实 OCR 服务可在后续接入。
- 新增人员后自动配置项目有效区域权限并尝试下发设备。人员尚未通过安全教育时，档案正常保存，设备下发保持待处理并返回具体原因。

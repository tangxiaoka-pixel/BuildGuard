# BuildGuard 设备厂家联调清单

## 联调前准备

- 确认服务器地址，例如 `https://buildguard.qivoria.com`。
- 在后台进入：项目级 -> 设备与门禁 -> 开放接入。
- 创建或启用接入账号，获取 `api_key` 和 `api_secret`。
- 确认设备能配置双发地址。
- 确认设备上传的 `client_serial` 是稳定唯一值。

## 第一阶段：连通性

- 调用 `Signin`，确认返回 `code=00`。
- 调用 `KeepAlive`，确认返回 `code=00`。
- 后台“开放接口调用日志”能看到请求和响应。
- 如果设备未提前建档，后台“待确认设备”能看到该 `client_serial`。

## 第二阶段：设备确认

- 在“待确认设备”点击“确认绑定”。
- 选择项目和区域。
- 确认后，设备进入“考勤设备”列表。
- 检查最近签到、最近心跳、最后 IP 是否更新。

## 第三阶段：考勤上传

- 调用 `UploadPassedLog` 上传一条进场记录。
- 调用 `UploadPassedLog` 上传一条出场记录。
- 后台“原始考勤数据”能看到原始报文。
- 项目级“考勤记录”能看到对应记录。
- “每日考勤明细”能按人员展示上下班时间和出勤时长。

## 第四阶段：异常验证

- 使用未知设备上传考勤，应保存为 `DEVICE_UNBOUND`。
- 使用不存在的人员上传考勤：
  - 有身份证号和姓名：自动创建待确认人员。
  - 无身份证号：保存原始考勤，标记 `UNMATCHED`。
- 使用错误签名调用接口，应返回 `code=25`。
- 使用过期 `timestamp` 调用接口，应返回 `code=31`。

## 厂家需要提供的信息

- 厂家名称
- 联系人和电话
- 设备品牌、型号
- `client_serial` 取值规则
- 上报字段样例
- 照片字段是否传 base64
- 心跳频率
- 网络出口 IP

## 平台确认项

- 接入账号状态为正常。
- 设备已绑定项目和区域。
- 项目中已经维护参建单位、班组、劳务人员。
- 如设备从人社平台同步人员，后台能看到自动创建的待确认人员。
- 如需要正式联调，建议单独创建测试项目和测试区域。
